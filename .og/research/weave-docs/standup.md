# Standup

> Quick summary for daily standups

The Standup report provides a condensed view of recent activity, designed for daily standup meetings.

## Who can see it?

* **Admin** and **Manager** roles can see standups for any team
* **Team lead** role can see standups for their teams
* **Member** role cannot access standup reports

## Default time range

Unlike other reports, the Standup report defaults to **this week** to show the most recent activity.

## What's included

* **Recent PRs** — Pull requests opened and merged this week
* **Recent reviews** — Code reviews given
* **Tasks completed** — Issues closed this week
* **Activity timeline** — Day-by-day breakdown

## Using in standups

The Standup report is designed to answer:

* What did the team accomplish yesterday?
* What's in progress today?
* Are there any blockers?

Scroll through team members to see individual contributions.
