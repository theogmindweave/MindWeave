## 3. Onboarding

### 3.1 Onboarding Playbook

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    ONBOARDING PLAYBOOK                                   │
│                    (14-Day Standard Onboarding)                          │
│                                                                         │
│   DAY 1: KICKOFF                                                        │
│   ─────────────────────────────────────────────────────────────────     │
│   □ Send welcome email with getting started guide                       │
│   □ Schedule kickoff call                                               │
│   □ Create customer success plan                                        │
│   □ Invite to customer Slack channel                                    │
│                                                                         │
│   DAY 1-3: KICKOFF CALL (60 min)                                        │
│   ─────────────────────────────────────────────────────────────────     │
│   □ Introductions and relationship building                             │
│   □ Review customer goals and success criteria                          │
│   □ Walk through implementation plan                                    │
│   □ Confirm stakeholders and communication preferences                  │
│   □ Schedule Week 1 training                                            │
│                                                                         │
│   DAY 3-5: TECHNICAL SETUP                                              │
│   ─────────────────────────────────────────────────────────────────     │
│   □ API key integration                                                 │
│   □ SSO configuration (if applicable)                                   │
│   □ Team member invitations                                             │
│   □ Initial data import/sync                                            │
│                                                                         │
│   DAY 5-7: TRAINING SESSION (60 min)                                    │
│   ─────────────────────────────────────────────────────────────────     │
│   □ Platform walkthrough                                                │
│   □ Key features for their use case                                     │
│   □ Hands-on exercises                                                  │
│   □ Q&A                                                                 │
│                                                                         │
│   DAY 7-10: GUIDED USAGE                                                │
│   ─────────────────────────────────────────────────────────────────     │
│   □ Check first data is flowing                                         │
│   □ Share "first insight" finding                                       │
│   □ Address questions via Slack                                         │
│   □ Monitor adoption metrics                                            │
│                                                                         │
│   DAY 10-14: GRADUATION CHECK-IN                                        │
│   ─────────────────────────────────────────────────────────────────     │
│   □ Review adoption against success criteria                            │
│   □ Identify any blockers                                               │
│   □ Transition to regular cadence                                       │
│   □ NPS survey                                                          │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 3.2 Onboarding Success Criteria

| Milestone | Target | Measurement |
|-----------|--------|-------------|
| Technical setup complete | Day 5 | API connected, SSO configured |
| Team onboarded | Day 7 | 80% of users logged in |
| First value moment | Day 7 | Customer identifies first insight |
| Full adoption | Day 14 | Using 3+ core features |
| Onboarding NPS | Day 14 | >8 score |

## 4. Health Scoring

### 4.1 Customer Health Score Model

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    CUSTOMER HEALTH SCORE                                 │
│                       (0-100 Scale)                                      │
│                                                                         │
│   COMPONENT          WEIGHT     METRICS                                 │
│   ─────────────────────────────────────────────────────────────────     │
│                                                                         │
│   PRODUCT USAGE       40%                                               │
│   ├── DAU/WAU ratio              (Target: >60%)           +15           │
│   ├── Feature breadth            (Target: 5+ features)    +10           │
│   ├── Data volume growth         (Target: +10% MoM)       +10           │
│   └── Session depth              (Target: >5 min)         +5            │
│                                                                         │
│   ENGAGEMENT          30%                                               │
│   ├── CSM meeting attendance     (Target: >80%)           +10           │
│   ├── Support ticket trend       (Target: Declining)      +5            │
│   ├── Training completion        (Target: >70%)           +10           │
│   └── NPS response               (Target: Promoter)       +5            │
│                                                                         │
│   OUTCOMES            20%                                               │
│   ├── Goals achieved             (Target: >80%)           +10           │
│   ├── ROI documented             (Target: Yes)            +5            │
│   └── Expansion potential        (Target: Identified)     +5            │
│                                                                         │
│   RELATIONSHIP        10%                                               │
│   ├── Executive sponsor          (Target: Engaged)        +5            │
│   └── Multi-threading            (Target: 3+ contacts)    +5            │
│                                                                         │
│   HEALTH THRESHOLDS                                                     │
│   ─────────────────────────────────────────────────────────────────     │
│   80-100: Healthy (Green)    - Standard monitoring                      │
│   60-79:  At Risk (Yellow)   - Proactive intervention                   │
│   0-59:   Critical (Red)     - Immediate escalation                     │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 4.2 Health Score Actions

| Score | Status | Actions |
|-------|--------|---------|
| 80-100 | Green | Growth focus, reference/case study ask |
| 60-79 | Yellow | Root cause analysis, recovery plan |
| 40-59 | Red | Executive escalation, intensive engagement |
| 0-39 | Critical | All hands, save or transition plan |

## 5. Renewal Management

### 5.1 Renewal Process

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    RENEWAL TIMELINE                                      │
│                                                                         │
│   -120 DAYS: HEALTH ASSESSMENT                                          │
│   ─────────────────────────────────────────────────────────────────     │
│   □ Review health score and trends                                      │
│   □ Analyze usage patterns                                              │
│   □ Document value delivered                                            │
│   □ Identify expansion opportunities                                    │
│                                                                         │
│   -90 DAYS: EXECUTIVE BUSINESS REVIEW                                   │
│   ─────────────────────────────────────────────────────────────────     │
│   □ Schedule EBR with key stakeholders                                  │
│   □ Present value achieved                                              │
│   □ Discuss roadmap and future needs                                    │
│   □ Introduce renewal discussion                                        │
│                                                                         │
│   -60 DAYS: RENEWAL PROPOSAL                                            │
│   ─────────────────────────────────────────────────────────────────     │
│   □ Send renewal proposal                                               │
│   □ Include expansion recommendations                                   │
│   □ Propose pricing (standard increase 3-5%)                            │
│   □ Schedule negotiation call                                           │
│                                                                         │
│   -30 DAYS: NEGOTIATION                                                 │
│   ─────────────────────────────────────────────────────────────────     │
│   □ Address objections                                                  │
│   □ Finalize terms                                                      │
│   □ Route for approval                                                  │
│   □ Send contract                                                       │
│                                                                         │
│   -14 DAYS: CLOSE                                                       │
│   ─────────────────────────────────────────────────────────────────     │
│   □ Obtain signature                                                    │
│   □ Process billing update                                              │
│   □ Celebrate and thank customer                                        │
│                                                                         │
│   0 DAYS: RENEWAL COMPLETE                                              │
│   ─────────────────────────────────────────────────────────────────     │
│   □ Update CRM                                                          │
│   □ Reset success plan                                                  │
│   □ Begin new cycle                                                     │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 5.2 Renewal Playbooks

| Scenario | Strategy | Owner |
|----------|----------|-------|
| Healthy + expansion | Bundle upsell with renewal | CSM + Sales |
| Healthy + flat | Smooth renewal, lock in multi-year | CSM |
| At risk | Recovery plan before renewal | CSM + Manager |
| Churn signal | Executive intervention, save offer | CS Leader |

## 6. Expansion & Upsell

### 6.1 Expansion Playbook

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    EXPANSION OPPORTUNITIES                               │
│                                                                         │
│   SEAT EXPANSION                                                        │
│   ─────────────────────────────────────────────────────────────────     │
│   Trigger: >80% seat utilization                                        │
│   Script: "I noticed your team is at capacity. As you bring on          │
│   more developers, we can add seats at $X/seat. Would you like          │
│   to discuss expanding your license?"                                   │
│   Commission: AE gets 5% of expansion                                   │
│                                                                         │
│   TIER UPGRADE                                                          │
│   ─────────────────────────────────────────────────────────────────     │
│   Trigger: Feature requests for higher tier                             │
│   Script: "You mentioned needing [feature]. That's available on         │
│   our Pro tier, which also includes [additional benefits]. Let me       │
│   show you the ROI of upgrading."                                       │
│   Commission: AE gets 10% of first year delta                           │
│                                                                         │
│   PROFESSIONAL SERVICES                                                 │
│   ─────────────────────────────────────────────────────────────────     │
│   Trigger: Complex implementation needs                                 │
│   Script: "For your custom requirements, our professional services      │
│   team can help with [specific deliverable]. This typically takes       │
│   [timeline] and costs [price]."                                        │
│   Commission: CSM gets 5% of services revenue                           │
│                                                                         │
│   MULTI-YEAR COMMITMENT                                                 │
│   ─────────────────────────────────────────────────────────────────     │
│   Trigger: Strong health, renewal approaching                           │
│   Script: "Given your success, I'd like to offer a multi-year           │
│   option with [discount] and price protection."                         │
│   Commission: Standard renewal commission + bonus                       │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 6.2 Expansion Signals

| Signal | Score Weight | Action |
|--------|--------------|--------|
| Seat utilization >80% | High | Proactive seat expansion offer |
| Feature requests for higher tier | High | Upgrade demo |
| New department asking questions | Medium | Multi-team expansion |
| Growing Claude usage | Medium | Usage-based expansion |
| Positive NPS + testimonial offer | High | Reference + case study |

### 7.2 Escalation Matrix
