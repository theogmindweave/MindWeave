## 2. Prospecting & Qualification

### 2.1 Target Account Identification

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    IDEAL ACCOUNT SIGNALS                                 │
│                                                                         │
│   STRONG FIT (Tier 1 - Prioritize)                                      │
│   ─────────────────────────────────────────────────────────────────     │
│   ✓ Using Claude API (mentioned in job posts, blogs, repos)             │
│   ✓ 10+ developers                                                      │
│   ✓ Recent funding (Series A+) or profitable                            │
│   ✓ Growing engineering team (hiring signals)                           │
│   ✓ Enterprise customers (need governance)                              │
│   ✓ Regulated industry (fintech, healthcare)                            │
│                                                                         │
│   MODERATE FIT (Tier 2 - Nurture)                                       │
│   ─────────────────────────────────────────────────────────────────     │
│   ~ Exploring AI/Claude (early stage)                                   │
│   ~ 5-10 developers                                                     │
│   ~ Startup without governance focus yet                                │
│   ~ Using competitors (OpenAI, etc)                                     │
│                                                                         │
│   POOR FIT (Tier 3 - Deprioritize)                                      │
│   ─────────────────────────────────────────────────────────────────     │
│   ✗ No Claude usage or plans                                            │
│   ✗ <5 developers                                                       │
│   ✗ No budget for tools                                                 │
│   ✗ Heavily invested in competitor                                      │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 2.2 BANT Qualification

| Criteria | Question | Qualifying Answer |
|----------|----------|-------------------|
| **Budget** | "Do you have budget allocated for developer tools?" | Yes, or can allocate |
| **Authority** | "Who else would be involved in this decision?" | Know the decision makers |
| **Need** | "What challenges do you face with Claude usage?" | Articulated pain |
| **Timeline** | "When are you looking to solve this?" | Within 3 months |

## 3. Demo Best Practices

### 3.1 Demo Structure

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    DEMO FLOW (30 MINUTES)                                │
│                                                                         │
│   OPENING (3 min)                                                       │
│   ─────────────────────────────────────────────────────────────────     │
│   • Confirm attendees and roles                                         │
│   • Recap discovery findings                                            │
│   • Set agenda and expectations                                         │
│   • "By the end, you'll see how MindWeave solves [their pain]"          │
│                                                                         │
│   DEMO (20 min)                                                         │
│   ─────────────────────────────────────────────────────────────────     │
│                                                                         │
│   Act 1: The Dashboard (5 min)                                          │
│   • Login flow (SSO emphasis if enterprise)                             │
│   • Overview metrics - immediate visibility                             │
│   • "In 30 seconds, you see what used to take hours"                    │
│                                                                         │
│   Act 2: Token Analytics (5 min)                                        │
│   • Usage trends and patterns                                           │
│   • Cost attribution by team                                            │
│   • Model comparison (Opus vs Sonnet vs Haiku)                          │
│   • "This is where customers find 30-40% savings"                       │
│                                                                         │
│   Act 3: MCP Registry (5 min)                                           │
│   • Discover all deployed tools                                         │
│   • Approval workflows                                                  │
│   • Security scanning                                                   │
│   • "Complete visibility into your AI tool stack"                       │
│                                                                         │
│   Act 4: Compliance (5 min)                                             │
│   • Audit log exploration                                               │
│   • Access controls                                                     │
│   • Export capabilities                                                 │
│   • "Meet your compliance requirements automatically"                   │
│                                                                         │
│   CLOSE (7 min)                                                         │
│   ─────────────────────────────────────────────────────────────────     │
│   • Summarize value seen                                                │
│   • Pricing overview                                                    │
│   • Propose next steps (trial, POC, security review)                    │
│   • Lock in timeline                                                    │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 3.2 Demo Do's and Don'ts

| Do | Don't |
|----|-------|
| Customize to their use case | Give a generic walkthrough |
| Pause for questions | Rush through features |
| Focus on 3-4 key capabilities | Show every feature |
| Use their terminology | Use jargon they don't know |
| Connect features to their pain | Demo features without context |
| Leave time for discussion | Run over time |

## 4. Objection Handling

### 4.1 Common Objections

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    OBJECTION HANDLING PLAYBOOK                           │
│                                                                         │
│   OBJECTION: "We can build this ourselves"                              │
│   ─────────────────────────────────────────────────────────────────     │
│   Response Framework:                                                   │
│   1. Acknowledge: "Many teams consider that initially"                  │
│   2. Question: "What's your team's capacity for internal tools?"        │
│   3. Reframe: "The question is: build vs buy opportunity cost"          │
│   4. Evidence: "Our customers found building takes 6+ months"           │
│   5. Close: "With MindWeave, you're live in 30 minutes"                 │
│                                                                         │
│   OBJECTION: "We're too early for governance"                           │
│   ─────────────────────────────────────────────────────────────────     │
│   Response Framework:                                                   │
│   1. Acknowledge: "Governance can sound heavy"                          │
│   2. Question: "When do you expect your AI usage to grow?"              │
│   3. Reframe: "Early visibility prevents later problems"                │
│   4. Evidence: "Teams that wait have 3x the cleanup cost"               │
│   5. Close: "Start with Team tier - grow when ready"                    │
│                                                                         │
│   OBJECTION: "Price is too high"                                        │
│   ─────────────────────────────────────────────────────────────────     │
│   Response Framework:                                                   │
│   1. Acknowledge: "Budget matters, let's understand the value"          │
│   2. Question: "What's your current Claude spend?"                      │
│   3. Reframe: "Average customer saves 35% on token costs"               │
│   4. Calculate: "At your spend, that's $X savings/month"                │
│   5. Close: "MindWeave often pays for itself in month 1"                │
│                                                                         │
│   OBJECTION: "We use OpenAI, not Claude"                                │
│   ─────────────────────────────────────────────────────────────────     │
│   Response Framework:                                                   │
│   1. Acknowledge: "Many teams use multiple models"                      │
│   2. Question: "Are you exploring Claude at all?"                       │
│   3. Reframe: "Multi-model is the future"                               │
│   4. Future: "We're expanding to other providers"                       │
│   5. Close: "Let's stay in touch as your stack evolves"                 │
│                                                                         │
│   OBJECTION: "Need to talk to my team/boss"                             │
│   ─────────────────────────────────────────────────────────────────     │
│   Response Framework:                                                   │
│   1. Acknowledge: "Of course, this is a team decision"                  │
│   2. Question: "Who would need to be involved?"                         │
│   3. Enable: "Can I send you a summary to share?"                       │
│   4. Expand: "Would it help to include them in a follow-up?"            │
│   5. Close: "Let's schedule a call with the broader team"               │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 4.2 Competitive Positioning

| Competitor | Their Strength | Our Counter |

## 5. Trial & POC Management

### 5.1 Trial Success Framework

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    TRIAL SUCCESS PLAYBOOK                                │
│                         14-Day Trial                                     │
│                                                                         │
│   DAY 1: KICKOFF                                                        │
│   ─────────────────────────────────────────────────────────────────     │
│   □ Welcome email with setup guide                                      │
│   □ 15-min onboarding call                                              │
│   □ Connect Claude API keys                                             │
│   □ Define success criteria (3-5 metrics)                               │
│   □ Schedule Day 7 check-in                                             │
│                                                                         │
│   DAY 3: EARLY VALUE                                                    │
│   ─────────────────────────────────────────────────────────────────     │
│   □ Check first data is flowing                                         │
│   □ Share "first insight" screenshot                                    │
│   □ Ensure key stakeholders have access                                 │
│   □ Address any technical blockers                                      │
│                                                                         │
│   DAY 7: MID-TRIAL CHECK                                                │
│   ─────────────────────────────────────────────────────────────────     │
│   □ 30-min progress call                                                │
│   □ Review usage patterns                                               │
│   □ Demonstrate value findings                                          │
│   □ Identify additional use cases                                       │
│   □ Confirm decision timeline                                           │
│                                                                         │
│   DAY 12: CLOSE PREP                                                    │
│   ─────────────────────────────────────────────────────────────────     │
│   □ Prepare trial summary report                                        │
│   □ Calculate ROI based on usage                                        │
│   □ Draft proposal                                                      │
│   □ Schedule closing call                                               │
│                                                                         │
│   DAY 14: CLOSE                                                         │
│   ─────────────────────────────────────────────────────────────────     │
│   □ Present trial results                                               │
│   □ Review proposal and pricing                                         │
│   □ Handle final objections                                             │
│   □ Get commitment or clear next steps                                  │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 5.2 Trial Success Metrics

| Metric | Good Trial | At Risk |
|--------|------------|---------|
| Daily active users | >50% of team | <20% of team |
| Features explored | 5+ features | 1-2 features |
| Stakeholders involved | 3+ | 1 (only champion) |
| Questions asked | Frequent | None |
| Success criteria | Met 3+ of 5 | Met <2 |

## 6. Closing Techniques

### 6.1 Closing Framework

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    CLOSING PLAYBOOK                                      │
│                                                                         │
│   TRIAL CLOSE (End of trial call)                                       │
│   ─────────────────────────────────────────────────────────────────     │
│   "Based on what you've seen in the trial, you mentioned [success       │
│   criteria]. We've demonstrated [specific value]. The Pro tier at       │
│   $2,000/month gives you [benefits]. Are you ready to move forward?"    │
│                                                                         │
│   SUMMARY CLOSE (After detailed evaluation)                             │
│   ─────────────────────────────────────────────────────────────────     │
│   "Let me recap: You need [need]. We've shown [capability].             │
│   The investment is [price] which you'll recoup through [ROI].          │
│   What else do you need to make a decision?"                            │
│                                                                         │
│   ASSUMPTIVE CLOSE (Strong buying signals)                              │
│   ─────────────────────────────────────────────────────────────────     │
│   "Great, so we're aligned on the Pro tier. Should I send over          │
│   the agreement for signature, or do you prefer we walk through         │
│   the terms on a call first?"                                           │
│                                                                         │
│   URGENCY CLOSE (Stalled deal)                                          │
│   ─────────────────────────────────────────────────────────────────     │
│   "I understand you're still evaluating. The challenge is, every        │
│   day without MindWeave is another day of untracked costs and           │
│   risk. What would it take to make a decision this week?"               │
│                                                                         │
│   NEXT STEP CLOSE (Not ready to buy)                                    │
│   ─────────────────────────────────────────────────────────────────     │
│   "It sounds like [blocker] is the main hold-up. Let's address          │
│   that directly. I'll set up a call with [relevant person] for          │
│   Thursday. Does 2pm work?"                                             │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 6.2 Deal Acceleration Tactics

| Tactic | When to Use | Script |
|--------|-------------|--------|
| Discount | End of quarter | "I can offer 15% off annual if we close by Friday" |
| Extended trial | Technical blocker | "Let's extend 1 week to address [issue]" |
| Executive intro | Stalled champion | "Would a call with our CEO help?" |
| Reference call | Trust building | "Can I connect you with [similar company]?" |
| Pilot scope | Too big to start | "Let's start with one team and prove value" |

## 7. Expansion & Upsell

### 7.1 Expansion Signals

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    EXPANSION SIGNALS                                     │
│                                                                         │
│   USAGE-BASED SIGNALS                                                   │
│   ├── Approaching seat limit (>80% utilization)                         │
│   ├── Token usage growing >20% MoM                                      │
│   ├── New teams/departments accessing                                   │
│   ├── MCP server count increasing                                       │
│   └── Feature exploration (new areas of product)                        │
│                                                                         │
│   RELATIONSHIP SIGNALS                                                  │
│   ├── Champion promoted                                                 │
│   ├── New stakeholders asking about product                             │
│   ├── Positive feedback in reviews                                      │
│   ├── Reference/testimonial willingness                                 │
│   └── Requesting features on higher tiers                               │
│                                                                         │
│   COMPANY SIGNALS                                                       │
│   ├── Funding round announced                                           │
│   ├── Hiring spike in engineering                                       │
│   ├── New AI/Claude initiatives announced                               │
│   ├── Enterprise customer wins (they need governance)                   │
│   └── Compliance requirements changing                                  │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 7.2 Upsell Playbook

| From | To | Trigger | Approach |
|------|------|---------|----------|
| Team | Pro | >15 seats needed | "You're growing! Pro gives you SSO and more seats" |
| Team | Pro | SSO request | "SSO is on Pro tier - let's upgrade" |
| Pro | Enterprise | >75 seats | "You've outgrown Pro - let's discuss Enterprise" |
| Pro | Enterprise | Custom needs | "Your requirements need Enterprise flexibility" |
