## 3Ô∏è‚É£ SECURITY ARCHITECTURE

### Data Protection

**In Transit (TLS 1.3):**
- All API calls encrypted
- All external integrations HTTPS
- No unencrypted data over network

**At Rest (AES-256):**
- Database encryption enabled
- Backup encryption enabled
- Key rotation (quarterly)
- Separate encryption keys per environment

**Key Management:**
- Keys stored in AWS Secrets Manager (or equivalent)
- No hardcoded secrets
- Rotation policy (90 days)
- Access logs (who accessed what key)

---

### Access Control (RBAC)

**Roles:**
- **Owner:** Full access (create, read, update, delete)
- **Member:** Use features, see usage
- **View-Only:** Read-only access

**Implementation:**
- Every API endpoint checks role
- Database row-level security
- Audit logs every access decision

---

### Audit Logs

**Logged Events:**
- User login/logout
- API calls (endpoint + parameters)
- MCP accessed/created/deleted
- Permission changes
- Export creation
- Compliance report generation

**Retention:**
- MVP: 3 months (free tier), 6 months (paid)
- Month 6: 6+ years (compliance requirement)

**Export:**
- CSV format (auditor-friendly)
- JSON format (programmatic access)
- Time range selector
- Filter by action/user

---

### Incident Response

**Plan:**
1. **Detect:** Automated alerts (suspicious activity)
2. **Assess:** Security team evaluates severity
3. **Contain:** Isolate affected systems
4. **Eradicate:** Fix root cause
5. **Recover:** Restore service + data
6. **Post-Mortem:** Document + improve

**Timeline:**
- P0 (Critical): Response within 1 hour
- P1 (High): Response within 4 hours
- P2 (Medium): Response within 24 hours
